# Инструкции по деплою

## Проблема, которая была решена

При деплое сайта медиа-файлы (фото и видео) не отображались из-за **некорректных имен файлов**:

1. **Кириллица в названиях**: `/video/Кубрик-3.0.mp4` и `.webm`
2. **Пробелы в названиях**: `containers-cropped/yang ming.png` и `port (1).webp`
3. **Специальные символы**: скобки в названии `port (1).webp`

Многие CDN и веб-серверы (особенно на Windows или при использовании URL encoding) не корректно обрабатывают такие символы.

## Что было исправлено

### Переименованные файлы:

**Видео файлы:**
- `Кубрик-3.0.mp4` → `hero-video.mp4`
- `Кубрик-3.0.webm` → `hero-video.webm`

**Изображения:**
- `port (1).webp` → `port-crane.webp`
- `yang ming.png` → `yang-ming.png`

### Обновленные компоненты:

1. **src/components/HeroSection.tsx** - обновлены пути к видео
2. **src/components/MarqueePartners.tsx** - обновлен путь к логотипу Yang Ming

## Как задеплоить

### Вариант 1: Загрузить новый dist.zip

Готовый архив `dist.zip` уже содержит все исправленные файлы и пути.

1. Удалите старые файлы с сервера
2. Загрузите и распакуйте новый `dist.zip`
3. Убедитесь, что корневой `index.html` находится в корне вашего веб-сервера

### Вариант 2: Пересборка и деплой вручную

```bash
# Пересобрать проект
npm run build

# Создать архив (опционально)
zip -r dist.zip dist/

# Загрузите содержимое папки dist/ на ваш хостинг
```

## Рекомендации для будущего

1. **Всегда используйте ASCII-символы** в названиях файлов
2. **Избегайте пробелов** - используйте дефисы или underscores
3. **Не используйте спецсимволы** типа `()[]{}` в названиях файлов
4. **Используйте lowercase** для названий файлов для лучшей совместимости

### Хорошие примеры названий:
- `hero-video.mp4`
- `company-logo.png`
- `user-avatar_2024.jpg`
- `service-image-01.webp`

### Плохие примеры:
- `Видео (1).mp4` ❌
- `Logo Company.PNG` ❌
- `Photo #123.jpg` ❌
- `файл-с-кириллицей.png` ❌

## Проверка после деплоя

После деплоя откройте сайт и проверьте:

1. ✅ Фоновое видео на главной странице загружается
2. ✅ Логотипы партнеров отображаются (особенно Yang Ming)
3. ✅ Все изображения в секциях загружаются
4. ✅ Откройте консоль браузера (F12) - не должно быть 404 ошибок

## Технические детали

- **Vite** автоматически копирует содержимое `public/` в `dist/` при сборке
- Пути в компонентах используют абсолютные пути от корня (например, `/images/...`)
- Видео-файлы большие (~17MB), убедитесь что они загружены полностью

## Контакты для поддержки

Если проблемы остались после деплоя:
1. Проверьте консоль браузера на наличие 404 ошибок
2. Убедитесь, что все файлы из `dist/` загружены на сервер
3. Проверьте права доступа к файлам на сервере
4. Убедитесь, что веб-сервер правильно настроен для SPA (Single Page Application)
