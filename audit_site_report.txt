                        2. ЧТО НЕ ХВАТАЕТ

КРИТИЧНО (необходимо реализовать):

2.1. ОТСУТСТВУЮЩИЕ СТРАНИЦЫ
────────────────────────────────────────────────────────────────────────────────
• /privacy — Политика конфиденциальности
  - Ссылка в Footer ведёт на несуществующую страницу
  - Требуется для соответствия законодательству о ПД (ФЗ-152)
  - Нужна для работы формы заявки

• /terms — Пользовательское соглашение
  - Рекомендуется для легитимности

• Детальные страницы услуг (отдельные URL)
  - /services/sea — Морские перевозки
  - /services/auto — Автоперевозки
  - /services/air — Авиадоставка
  - /services/customs — Таможенное оформление
  - Улучшит SEO и пользовательский опыт

2.2. НЕДОСТАЮЩИЙ КОНТЕНТ
────────────────────────────────────────────────────────────────────────────────
• Калькулятор стоимости
  - Главная фича для логистического сайта
  - Позволит пользователю получить оценку без звонка
  - Увеличит конверсию

• Реальные фотографии компании/офиса
  - Используются stock фото (Unsplash) для сотрудников
  - Нет фото офиса, команды, процессов

• Сертификаты и лицензии
  - Нет раздела с документами компании
  - Критично для B2B клиентов

• Блог/Новости
  - Отсутствует раздел актуальных материалов
  - Важно для SEO и экспертности

• Детальная информация о тарифах
  - Только примерные цены "от $XXX"
  - Нет прозрачной тарифной сетки

2.3. ФУНКЦИОНАЛЬНЫЕ ПРОБЕЛЫ
────────────────────────────────────────────────────────────────────────────────
• Форма не отправляет данные
  - Контактная форма только симулирует отправку (setTimeout)
  - Нет интеграции с бэкендом/почтой

• Отсутствует трекинг груза
  - Заявлено "онлайн-мониторинг грузов"
  - Функция не реализована

• Нет личного кабинета
  - Для постоянных клиентов логично иметь ЛК

• Нет чата поддержки
  - Заявлена поддержка 24/7, но нет виджета чата

• Карта в Contacts не работает
  - Iframe с Google Maps содержит некорректные координаты (заглушка)

2.4. НЕДОСТАЮЩИЕ SEO ЭЛЕМЕНТЫ
────────────────────────────────────────────────────────────────────────────────
• Нет OG-изображения (/og-image.jpg)
  - Файл указан в SEO компоненте, но не существует

• Нет favicon.ico
  - Только favicon.svg, нужен fallback

• Robots.txt не содержит ссылку на sitemap
  - Нужно добавить: Sitemap: https://armax-logistics.com/sitemap.xml

• Нет hreflang для других языков
  - Если планируется многоязычность

• Нет микроразметки FAQ
  - На странице FAQ нет FAQPage schema

3.1. UX/UI УЛУЧШЕНИЯ
────────────────────────────────────────────────────────────────────────────────
• Мобильное меню
  - При открытии нет анимации (резкое появление)
  - Нет блокировки скролла body при открытом меню
  - Не закрывается по клику вне меню

• Hero секция
  - На мобильных устройствах статистика мелкая
  - Кнопка "Наши услуги" только на мобильных (нет на десктопе)
  - Фоновое видео тяжёлое (требует оптимизации)

• Формы
  - Нет валидации в реальном времени
  - Нет маски для телефона (+7 (___) ___-__-__)
  - Нет автозаполнения (autocomplete атрибуты)
  - Email поле необязательное, но нужно для связи

• Навигация
  - Нет хлебных крошек на страницах (компонент есть, но не используется)
  - Нет индикатора текущего раздела в Footer

• Таблицы с данными
  - Маршруты в Geography могли бы быть фильтруемыми
  - FAQ фильтрация работает, но сброс неочевиден

• Интерактивная карта
  - Инструкции внизу карты мелкие
  - Мобильное управление (pinch-zoom) не оптимизировано
  - Нет кнопки "показать мой маршрут"

3.2. СТРУКТУРА И КОНТЕНТ
────────────────────────────────────────────────────────────────────────────────
• Страница Services
  - Дублирование: интерактивные табы + сетка услуг показывают одно и то же
  - Можно убрать нижнюю сетку или сделать её менее подробной

  - Timeline можно сделать интерактивным
  - Нет фото команды

  - Отзывы выглядят "постановочными" (все 5 звёзд)

  - Нет возможности оставить отзыв

• Footer
  - Колонка "Услуги" — неактивные ссылки (просто текст)

3.3. АДАПТИВНОСТЬ
────────────────────────────────────────────────────────────────────────────────

  - На md разрешении большая карточка занимает всё

                       5. ЧТО ОПТИМИЗИРОВАТЬ
════════════════════════════════════════════════════════════════════════════════

5.1. ПРОИЗВОДИТЕЛЬНОСТЬ
────────────────────────────────────────────────────────────────────────────────
• Изображения:
  - /images/customs.png — 54KB PNG, можно конвертировать в WebP
  - Нет lazy loading на всех изображениях (только на партнёрах)
  - Нет srcset/sizes для адаптивных изображений
  - Отзывы загружают фото с Unsplash (внешний домен, задержка)

• Видео:
  - Hero видео загружается сразу (preload="auto")
  - Файл большой, лучше preload="metadata" или preload="none"
  - Нужен постер пока грузится

• Шрифты:
  - Google Fonts загружается из внешнего источника
  - Рекомендуется self-hosted + font-display: swap

• JavaScript:
  - Все страницы загружаются сразу (нет code splitting)
  - React.lazy + Suspense для маршрутов

• CSS:
  - Tailwind очищает неиспользуемые стили (purge)
  - Но много компонентов shadcn/ui добавляют лишний CSS

• Карта:
  - TopoJSON грузится с CDN каждый раз
  - Можно закешировать или включить в бандл
  - MapGeography обёрнут в memo — хорошо

5.2. CORE WEB VITALS
────────────────────────────────────────────────────────────────────────────────
• LCP (Largest Contentful Paint):
  - Hero секция с видео может быть медленной
  - Рекомендуется preload для постера

• CLS (Cumulative Layout Shift):
  - Изображения без заданных размеров могут вызывать сдвиги
  - Добавить width/height или aspect-ratio

• FID/INP (Interactivity):
  - Много hover эффектов, но лёгкие
  - Карта может быть тяжёлой при первом взаимодействии

5.3. SEO ОПТИМИЗАЦИЯ
────────────────────────────────────────────────────────────────────────────────
• HTML lang атрибут
  - index.html: lang="en" — должно быть lang="ru"

• Заголовки:
  - Каждая страница имеет один H1 — хорошо
  - Иерархия H1 > H2 > H3 соблюдается — хорошо

• Meta теги:
  - theme-color: "#3b82f6" (синий) не соответствует бренду (coral)
  - Должен быть: #E85D3E

• Structured Data:
  - Нет FAQPage schema для /faq
  - Нет Service schema для услуг
  - breadcrumbSchema определён, но не используется

8.1. ДОСТУПНОСТЬ (ACCESSIBILITY)
────────────────────────────────────────────────────────────────────────────────
Найдено aria-* атрибутов: 28 (в основном в UI компонентах)
Найдено alt атрибутов: 7

Проблемы:
• Многие интерактивные элементы без role и aria-label
• Кнопки в карточках (BentoGrid) — div с onClick, не <button>
• FAQ аккордеон кастомный — нет aria-expanded
• Карта без альтернативного текста для screen readers
• Изображения в BentoGrid без alt (backgroundImage)
• Кнопки фильтров в FAQ без aria-pressed
